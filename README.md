# Caderno Temático: Programação Orientada a Objetos com NotebookLM

## Sobre o projeto
Este projeto foi desenvolvido como parte de um desafio da plataforma DIO.

O objetivo foi utilizar o NotebookLM como uma ferramenta de aprendizagem
ativa para estudar Programação Orientada a Objetos, também conhecida como POO.

Durante o projeto, foram selecionadas fontes de estudo sobre o tema, elaborados
prompts estratégicos e organizados os principais conceitos aprendidos em um
miniguia de estudos.

## Contexto e objetivos
O tema escolhido para o caderno temático foi Programação Orientada a Objetos.

Escolhi esse assunto porque estou iniciando meus estudos na área de
desenvolvimento de software e considero importante compreender como classes,
objetos, atributos e métodos são utilizados na construção de sistemas.

Os principais objetivos deste estudo foram:

- Compreender o conceito de Programação Orientada a Objetos;
- Entender a diferença entre classe e objeto;
- Aprender os conceitos de encapsulamento, herança, polimorfismo e abstração;
- Identificar exemplos práticos de POO;
- Criar um material que possa ser utilizado em futuras revisões.

## Engenharia de prompts
Durante o estudo, foram elaboradas diferentes perguntas para obter explicações,
comparações e exemplos práticos sobre POO. Na sua grande parte, pesquisas realizadas pelo próprio
LLM na aba "Deep Research"

## Dificuldades encontradas e aprendizados
Em alguns dos primeiros testes, os prompts utilizados eram muito genéricos.
Como resultado, o NotebookLM apresentou respostas corretas, mas pouco
aprofundadas.

Um exemplo foi o prompt:

> Explique POO.

Para melhorar a resposta, reformulei a pergunta:

> Explique os quatro pilares da Programação Orientada a Objetos separadamente,
> utilizando exemplos simples em Java.

A nova versão produziu uma resposta mais organizada e adequada ao meu nível de
conhecimento.

Esse processo demonstrou que prompts mais específicos tendem a gerar respostas
mais úteis para o estudo.
